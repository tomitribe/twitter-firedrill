In addition to the entities attribute in the JSON of a tweet (as documented in Entities in Twitter Objects), we also support extended_entities.  extended_entities is a collection of multiple media objects referenced in a tweet (sample output below). 

extended_entities represent multiple media entities in the text of the tweet as a single URL. Note that the ranges for these entities overlap, meaning that multiple images are represented by a single link. (Previously, each image had a text range/url.) Clients that do not expect overlapping ranges need to support extended_entities and multiple media tweets. The format of media entities remains unchanged except for this overlap.

Currently, the only type exposed in our entities is photo. We reserve the right to use extended_entities to expand the types of media we expose. Developers should use the type attribute from the media exposed to validate you are handling the types properly.

POSTing multiple media files

Uploading multiple media files is a 2 step process.

  1. First, use POST multipart/form-data or base64 encoded files to https://upload.twitter.com/1.1/media/upload.json.
  2. Second, collect the resulting IDs to pass as a comma separated list of media_ids for all the files you want to attach to your POST statuses / update.

Uploaded media files will be available for attachment for 60 minutes before they are flushed from the servers (if not attached to a tweet). Note that you need to call the upload endpoint once per media file you want to upload (so to upload three media files, call the upload endpoint three times, storing the media_ids ready to be attached to your POST statuses / update).

Fallback for older Clients/API Users

If your application falls back to entities if you can not support extended_entities, your application should be unaffected. The legacy entities is guaranteed to contain at most one single entry in its array, while the new extended_entities will contain an array of all entities associated with a tweet, up to a maximum of 28. Entities copied from other tweets will contain a source_status_id field with the ID of the original tweet.
